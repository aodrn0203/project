# 허프만 코딩 텍스트 압축기
유니코드(UTF-8) 문자 단위로 동작하는 허프만 코딩 기반 텍스트 압축 프로그램입니다.
한글·이모지 등 멀티바이트 문자를 코드포인트 단위로 처리하며, 압축 후 복원하여 원본과 일치하는지 검증합니다.

```
컴파일: gcc main.c -o huffman
실행:   ./huffman.exe
```
---
## 상수 정의
| 상수 | 값 | 설명 |
|---|---|---|
| `MAX_TEXT` | `10000` | 입력 텍스트 최대 바이트 길이 |
| `MAX_SYMBOLS` | `4096` | 서로 다른 문자(코드포인트) 최대 개수 |
---
## 자료구조
### Node — 허프만 트리 노드
```c
typedef struct Node {
    uint32_t ch;        // 코드포인트 (잎 노드에서만 의미)
    int freq;           // 빈도 (또는 두 자식 빈도의 합)
    int seq;            // 생성 순서 (tie-break 용)
    struct Node *left;  // 왼쪽 자식 (코드 0)
    struct Node *right; // 오른쪽 자식 (코드 1)
} Node;
```

| 필드 | 설명 |
|---|---|
| `ch` | 문자의 코드포인트. 잎(leaf) 노드에서만 의미가 있다 |
| `freq` | 등장 빈도. 내부 노드는 두 자식 빈도의 합 |
| `seq` | 생성 순서. 빈도가 같을 때 순서를 결정해 항상 같은 트리가 나오게 한다 |
| `left` / `right` | 왼쪽(코드 0) / 오른쪽(코드 1) 자식 |

### MinHeap — 최소 힙(우선순위 큐)
```c
typedef struct {
    Node* data[MAX_SYMBOLS * 2]; // 잎 n개 + 내부 n-1개 < 2n
    int size;
} MinHeap;
```
빈도가 작은 노드를 먼저 꺼낸다. 빈도가 같으면 `seq`가 작은 노드를 우선한다.

## 처리 흐름
| 단계 | 내용 |
|---|---|
| 0 | UTF-8 텍스트를 코드포인트 배열로 디코딩 |
| 1 | 빈도 분석 — 서로 다른 코드포인트와 등장 횟수 수집 |
| 2 | 허프만 트리 생성 (최소 힙 사용) |
| 3 | 코드표 생성 — 뿌리→잎 경로로 0/1 코드 부여 |
| 4 | 실제 압축 — 코드를 이어붙여 비트로 채움 |
| 5 | 결과 출력 (메뉴 선택) |
---
## 주요 함수
### UTF-8 변환
| 함수 | 설명 |
|---|---|
| `utf8Decode` | 문자열 앞 한 글자를 코드포인트로 디코딩, 차지한 바이트 수 반환 |
| `utf8Encode` | 코드포인트 하나를 UTF-8 바이트열로 인코딩, 쓴 바이트 수 반환 |

### 트리 / 힙
| 함수 | 설명 |
|---|---|
| `createNode` | 노드 하나를 새로 생성 |
| `isLeaf` | 잎 노드 여부 검사 (자식이 둘 다 없으면 잎) |
| `nodeLess` | 노드 비교 — 빈도 우선, 동률이면 `seq` 우선 |
| `heapPush` | 힙에 노드 삽입 (상향 정렬) |
| `heapPop` | 힙에서 최소 노드 추출 (하향 정렬) |
| `buildHuffmanTree` | 잎 생성 → 최소 두 노드 병합 반복 → 뿌리 반환 |

### 코드표 / 압축
| 함수 | 설명 |
|---|---|
| `generateCodes` | 뿌리에서 잎까지 내려가며 왼쪽=0, 오른쪽=1 코드 부여 |
| `findSym` | 코드포인트의 심볼 목록 인덱스 검색 |
| `freeTree` | 트리 메모리 재귀 해제 |

### 출력
| 함수 | 출력 섹션 |
|---|---|
| `printChar` | 한 문자 출력 (공백·줄바꿈·탭은 이름으로 표기) |
| `printSectionTable` | [1] 문자별 빈도 & 허프만 코드표 |
| `printSectionPacked` | [2] 실제 압축된 데이터 (16진수 바이트) |
| `printSectionAnalysis` | [3] 압축 결과 분석 (압축률 등) |
| `printSectionDecoded` | [4] 복원 결과 + 원본 일치 검증 |
| `runHuffman` | 텍스트 하나에 대한 전체 과정 수행 |
| `main` | 메인 메뉴 루프 / 텍스트 입력 처리 |

## tie-break 규칙
같은 입력이면 항상 같은 트리·코드가 나오도록 노드 비교 기준을 정해 둔다.

| 우선순위 | 기준 | 설명 |
|---|---|---|
| 1 | 빈도(`freq`) | 빈도가 작은 노드를 먼저 꺼낸다 |
| 2 | 생성 순서(`seq`) | 빈도가 같으면 `seq`가 빠른 노드 우선 |

잎 노드의 `seq`는 **코드포인트 오름차순 순위**로 정해, 입력 순서와 무관하게 결과가 동일하다.

## 사용 방법
1. 프로그램 실행 후 메인 메뉴에서 `1` (새 텍스트 압축하기) 선택
2. 텍스트 입력 — 마지막 줄에 점(`.`) 하나만 입력하면 종료
3. 결과 메뉴에서 보고 싶은 섹션 선택

| 입력 | 동작 |
|---|---|
| `1` | 문자별 빈도 & 허프만 코드표 |
| `2` | 실제 압축된 데이터 (16진수) |
| `3` | 압축 결과 분석 |
| `4` | 복원(압축 해제) 결과 |
| `0` | 메인 메뉴로 돌아가기 |
---
## 특징
- **유니코드 지원** — UTF-8을 코드포인트 단위로 처리해 한글·이모지도 정상 압축
- **단일 문자 처리** — 문자가 한 종류뿐이면 코드를 `"0"`으로 지정
- **복원 검증** — 압축 해제 후 원본과 100% 일치 여부 확인
- **윈도우 대응** — `SetConsoleOutputCP(CP_UTF8)`로 콘솔 한글 깨짐 방지
